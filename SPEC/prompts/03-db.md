STEP 3 — Normalizzazione DB (Mongo) per Dashboard

Contesto:
- I dati raw sono già stati importati in Mongo tramite STEP 2 (collection `reports_raw`).
- Ora devi creare una versione “normalized” stabile per alimentare backend API e dashboard.

Obiettivo:
Implementa un comando/script Node (es. `npm run ingest:normalize`) che:
1) legge i documenti presenti in `reports_raw`
2) per ciascun documento costruisce un documento in `reports_normalized` con uno schema minimo e coerente
3) scrive/aggiorna in modo idempotente (nessun duplicato se rilanciato)

Collection `reports_raw` (input) — schema minimo atteso:
- `filename` (string)
- `report_prefix` ("enterprise" | "users")
- `ingested_at` (ISO string o Date)
- `payload` (object)

Collection `reports_normalized` (output) — schema minimo richiesto:
- `source_filename` (string)  // riferimento a `reports_raw.filename`
- `report_prefix` ("enterprise" | "users")
- `report_start_day` (string | null) // se presente nel payload
- `report_end_day` (string | null)   // se presente nel payload
- `normalized_at` (ISO string o Date)
- `metrics` (object) // SOLO campi utili per la dashboard

Regole:
- Deduplica / upsert: la chiave unica è `source_filename`.
  Se esiste già un documento normalized per quel file, fai update (upsert).
- Resilienza schema: il payload dei report può cambiare.
  Quindi:
  - salva sempre `report_start_day` e `report_end_day` se presenti, altrimenti null
  - in `metrics` includi SOLO i campi che esistono (nessun campo obbligatorio oltre a quelli sopra)
  - non crashare se mancano campi: logga e continua
- Mantieni KISS: niente pipeline complesse, niente ORM, niente overengineering.

Cosa deve finire in `metrics` (minimo):
- `kpis` (object) con i principali contatori disponibili nel payload (anche se parziali)
- `timeseries` (array) se nel payload esistono dati temporali (opzionale)
- `top_users` (array) se nel payload esistono dati per utente (opzionale)

Alla fine:
- dimmi i file/moduli creati o modificati
- dimmi il comando esatto per normalizzare (esempi di `MONGODB_URI`)
- dimmi una verifica rapida (query mongo o endpoint) per controllare quanti documenti normalized sono stati creati
